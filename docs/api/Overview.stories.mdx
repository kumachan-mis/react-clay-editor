# Overview

The API of React Clay Editor consists of multiple components. The components can be grouped by fixed or scrollable, and
a container or a content.

<div style={{ display: 'flex', justifyContent: 'center' }}>
  <ApiExplanation colorized="all" />
</div>

<dl>
  <dt>
    <p>
      EditorRoot/ViewerRoot (<span style={{ color: colorPalette.fixedContainer }}>■</span> container of fixed contents)
    </p>
  </dt>
  <dd>
    <p>
      The root containers of the editor and the viewer. They receive properties used in the whole of the components.
    </p>
  </dd>
  <dt>
    <p>
      EditorSyntaxMenu (<span style={{ color: colorPalette.fixedContent }}>■</span> fixed content)
    </p>
  </dt>
  <dd>
    <p>
      A set of buttons and dropdown menus to apply formats interactively. It is not scroll out even a text field is
      scrolled.
    </p>
  </dd>
  <dt>
    <p>
      EditorTextFieldRoot/ViewerTextFieldRoot (<span style={{ color: colorPalette.scrollableContainer }}>■</span>
      container of scrollable contents)
    </p>
  </dt>
  <dd>
    <p>The containers of scrollable text fields. They grow taller until they fill up the space.</p>
  </dd>
  <dt>
    <p>
      EditorTextFieldHeader/ViewerTextFieldHeader (<span style={{ color: colorPalette.scrollableContent }}>■</span>
      scrollable content)
    </p>
  </dt>
  <dd>
    <p>The components displaying uneditable headers. They are useful to show a fixed title of a document.</p>
  </dd>
  <dt>
    <p>
      EditorTextFieldBody/ViewerTextFieldBody (<span style={{ color: colorPalette.scrollableContent }}>■</span>
      scrollable content)
    </p>
  </dt>
  <dd>
    <p>
      The components displaying main texts and controling interactions from a user. Users may scroll them vertically to
      view and edit their text.
    </p>
  </dd>
</dl>
